## 🔗 协作流程

### 前后端接口对齐

**登录认证接口**:
- `POST /api/auth/login` - 登录
- `POST /api/auth/logout` - 登出
- `GET /api/auth/session` - 获取 session

**模型接口**:
- `POST /api/models/add` - 添加模型

**技能商店接口**:
- `GET /api/skills/store/list` - 获取技能列表
- `POST /api/skills/install` - 安装技能
- `POST /api/skills/uninstall` - 卸载技能

### Git 分支策略

- `main` - 主分支
- `feature/login-auth` - 登录认证
- `feature/add-model` - 模型添加
- `feature/skill-store` - 技能商店
- `refactor/i18n` - 国际化重构

### 代码审查流程

1. 开发者创建功能分支
2. 完成开发后提交 PR
3. `architect` 进行代码审查
4. 通过审查后合并到 main
