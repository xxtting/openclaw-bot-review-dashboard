# 架构师任务分配

**角色**: 架构师 (云图)
**负责人**: architect agent
**任务类型**: 架构设计 + 代码审查

---

## 🎯 核心任务

### 1. 登录认证架构设计

**目标**: 设计安全、可扩展的登录认证系统

**技术选型**:
- [ ] 评估 `next-auth` vs 自实现 JWT
  - Pros/Cons 分析
  - 性能对比
  - 安全性评估
- [ ] 确定技术方案并记录决策依据
- [ ] 设计 API 接口规范
  - 登录接口 (`POST /api/auth/login`)
  - 登出接口 (`POST /api/auth/logout`)
  - Session 查询接口 (`GET /api/auth/session`)
- [ ] 设计中间件保护策略
  - 哪些路由需要保护
  - 未授权处理逻辑
- [ ] 安全性考虑
  - Session 过期策略
  - Token 存储方式
  - CSRF 防护

**交付物**:
- 架构设计文档 (`ARCHITECTURE.md`)
- API 接口规范 (`API-SPEC.md`)
- 技术选型报告

---

### 2. API 接口设计评审

**目标**: 确保前后端接口一致、规范

**任务**:
- [ ] 审核登录认证接口设计
- [ ] 审核模型添加接口设计
- [ ] 审核技能商店接口设计
- [ ] 统一错误码规范
- [ ] 统一响应格式

---

### 3. 代码审查

**目标**: 确保代码质量、架构一致性

**PR 审查清单**:
- [ ] 代码风格符合规范
- [ ] 类型安全（TypeScript）
- [ ] 错误处理完善
- [ ] 安全性检查
- [ ] 性能考虑
- [ ] 测试覆盖

**审查流程**:
1. 开发者提交 PR
2. Review 代码
3. 提出修改意见（如有）
4. 批准合并
